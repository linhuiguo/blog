## 一、已验证有效的写法（有真实数据支撑）

**案例**：GPT-5.5 保姆级教程（首篇头条教程）
- 数据：展现 3957 / 阅读 501 / 点赞 13 / 评论 1（头条新号起步，算不错）
- 结论：**步骤清晰、小白能懂**的写法能跑出有效数据，值得复用

**核心结构（照搬）**：
1. **标题钩子**：用"白嫖/免费/最强/不用花一分钱"等词 + 具体利益点 + `（保姆级教程）` 后缀
   - 例：`微软良心！不用花一分钱，把 GPT-5.5 接到你电脑上（保姆级教程）`
   - 例：`腾讯 Hy3 免费放了！手把手教你接到 Hermes / Codex / Kilo Code，小白也能白嫖`
2. **开头一句话说清"这东西强在哪 / 为什么免费"**：用真实测评数据（盲测排名、幻觉率、参数规模）建立可信度
3. **分路线**：按读者水平拆
   - 路线一「懒人版」：最少操作，装好就能用（如 Hermes Agent 内置模型）
   - 路线二「折腾版」：拿 API 配到自有工具（OpenRouter → Codex / Kilo Code / Cherry Studio / OpenClaw / Cline）
4. **每步写清"点哪、选啥、填啥"**，不跳步、不假设你懂
   - 例：`点齿轮 → 改中文 → 选模型 Hy3 → 点应用`
5. **关键配置直接给，照抄不猜**：
   - Base URL / API 端点
   - API Key（教在哪创建，提醒只显示一次）
   - Model / 模型名（必须真实，发布前 curl 或查官方确认，不瞎编）
6. **结尾加 FAQ + 博客引流**：`博客更多 AI 白嫖教程见 0149.cc.cd`

## 二、工具接入的通用范式（任何 OpenAI 兼容接口都适用）

所有支持"自定义 OpenAI 兼容端点"的工具，配置都是这三样：

| 配置项 | 填什么 |
|--------|--------|
| Base URL / API 地址 | 服务商给的 `https://.../v1` |
| API Key / 密钥 | 后台创建的 `sk-...` |
| Model / 模型名 | 服务商真实的模型 ID |

已实测过的免费模型：
- **腾讯 Hy3**：OpenRouter 模型 ID `tencent/hy3:free`，端点 `https://openrouter.ai/api/v1`（免费档有时效，约两周）
- **Windows Copilot 反代**：本地 `http://127.0.0.1:8080/v1`，模型名 `gpt-5.5` / `copilot`（仅文本，无图/视频）

接入时必须包含的实操段（用户硬性要求）：
- **配置到 Hermes Agent**：改 `~/.hermes/config.yaml` 的 `custom_providers` 加本地/远程基址，重启选模型
- **配置到 Codex**：设环境变量 `OPENAI_BASE_URL` / `OPENAI_API_KEY` / `OPENAI_MODEL`（不同版本变量名可能略异，逻辑一致）

## 三、发布流程（头条号「先生靠谱」）

脚本：`auto-video-edit/toutiao_post.py`（连 Edge CDP 端口 9222）
- 填标题 → 填正文（ProseMirror 粘贴 HTML）→ 滚到底 → 选无封面（force=True）→ 预览并发布 → 确认发布
- **只填不发布**：用 `fill_only.py` 填进编辑器后停住，留给人工加插图/修改
- CDP 浏览器拉起命令：`msedge --remote-debugging-port=9222 --user-data-dir="C:/Users/guo12/edge-debug"`

## 四、避坑清单

- ❌ 标题作者别用「然惊半」——头条号署名是「先生靠谱」（曾写错被纠正）
- ❌ 模型名/端点不能瞎编——发布前实测确认（曾靠 curl OpenRouter 验证 `tencent/hy3:free`）
- ❌ 不要对训练数据截止后的事件做绝对判断（如"GPT-5.5 不存在"）——可能是知识盲区
- ⚠️ 免费档有实效（Hy3 约两周），教程里别写死具体到期日，写"趁早用"
- ⚠️ 复杂开发任务免费小模型有差距，教程里如实说"适合日常任务"，不吹过头
